Dashboard Maintenance Guide
Step-by-step administrative guide for maintaining and updating the dashboard
1. Export Data
2. Format File
3. Upload
4. Verify
5. Archive

Step 1: Export Monthly Data from WMS

At the end of each month, export the following reports from the Holisol WMS system:

  • Inbound/GRN data (Plan vs Actual)
  • Outbound shipment details
  • Stock flow (Opening, Closing, Movement)
  • Quality issues and damage reports
  • Freight and transportation logs
  • Co-packing orders and production
  • Temperature monitoring data
  • Costing and invoice details

Step 2: Format the Source File

Ensure the exported file follows the standard template structure:

  • File format: .xlsx (Excel Workbook)
  • Sheet naming must match the standard convention (e.g., "20. Stock Flow", "15. Costing_Det")
  • Column headers must remain unchanged
  • New monthly data should be appended as new rows
  • Do not modify or delete historical data
  • Date format: MMM-YY (e.g., Jan-25)

Step 3: Upload to Dashboard

Click the Upload Data button in the top navigation bar. Drag and drop or browse for the formatted .xlsx file. The system will validate the file structure before processing.

Upon successful upload, all KPI cards, charts, and tables will automatically refresh with the new data. No manual UI changes are required.

Step 4: Verify & Archive

After upload, verify the following across all tabs:

  • Executive KPI cards show updated values
  • All chart trend lines include the new month
  • Tables display the latest records
  • Filter dropdowns include new period options

Archive the previous month's source file in the designated backup folder for audit compliance.

!
Important: Always back up the current dashboard_data.json and dashboard_data2.json files before uploading new data. The dashboard reads from these two JSON files located in the same directory as this HTML file.
